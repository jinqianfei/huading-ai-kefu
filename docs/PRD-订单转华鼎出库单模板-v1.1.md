## 一、业务背景

### 1.1 现状痛点

| 痛点 | 现状描述 | 影响 |
|------|----------|------|
| **订单格式不统一** | 客户上传的订单Excel格式各异，无法直接导入华鼎系统 | 需人工整理，耗时长 |
| **字段映射复杂** | 客户订单字段与华鼎模板31字段需要一一对应 | 易出错，效率低 |
| **SKU匹配繁琐** | 需在系统中查找每个商品的SKU编码 | 耗时长，易遗漏 |
| **门店信息分散** | 门店收货信息需从多个数据源获取 | 数据不一致 |

### 1.2 解决方案

**产品名称**：订单转华鼎出库单模板  
**产品形态**：AI客服Skill  
**核心价值**：客户上传订单Excel，自动转换为华鼎系统可导入的出库单模板

---

## 二、产品概述

### 2.1 产品定位

| 项目 | 内容 |
|------|------|
| **产品名称** | 订单转华鼎出库单模板 |
| **产品形态** | AI客服Skill（skill-order-to-huading-template） |
| **核心价值** | 将客户订单Excel一键转换为华鼎系统可导入的31字段出库单模板 |

### 2.2 目标用户

| 用户角色 | 使用场景 |
|----------|----------|
| 结算专员 | 日常接收客户订单，生成华鼎模板 |
| 客服组长 | 处理异常订单或批量订单 |

### 2.3 输入输出

| 项目 | 说明 |
|------|------|
| **输入** | 客户订单Excel文件 + 货主ID (shipper_id) |
| **输出** | 华鼎出库单模板Excel文件（31字段） |

### 2.4 输入数据格式

客户订单Excel格式示例（行号仅供参考）：

| 列 | 字段 | 示例值 |
|----|------|--------|
| A | 序号 | 1, 2, 3... |
| B | 商品编码 | P001 |
| C | 商品名称 | 农夫山泉550ml |
| D | 规格 | 规格：24瓶装 |
| E | 单位 | 件 |
| F | 数量 | 10 |
| G | 备注 | 需轻拿轻放 |

**抬头信息**（前几行）：

| 行 | 列B | 示例值 |
|----|------|--------|
| 1 | 订单编号 | DH-O-20260423-294092 |
| 2 | 门店名称 | 河北-石家庄永辉超市 |
| 3 | 收货人 | 张三 |
| 4 | 联系电话 | 13800138000 |
| 5 | 收货地址 | 河北省石家庄市XX区XX路 |

**解析规则**：
- 订单号：第1行B列
- 门店名：第2行B列（可能带"河北-"前缀，需清理）
- 商品明细：从第6行开始（A列有序号）
- 备注：从第7列(G列)提取，**不是**从规格列(D列)提取

## 三、功能详情

### 3.1 处理流程

```
┌─────────────────────────────────────────────────────────────────┐
│  Step 1: 订单解析                                                │
│  输入：客户订单Excel                                            │
│  输出：订单号、门店名、商品明细                                  │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  Step 2: 门店匹配                                               │
│  输入：门店名称                                                 │
│  处理：查询store_list表，获取门店编号、仓库、收货人等           │
│  输出：门店信息                                                 │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  Step 3: SKU匹配                                                │
│  输入：商品名称                                                 │
│  处理：查询shipper_sku_mapping表，获取系统SKU编号               │
│  输出：SKU匹配结果 + 未匹配商品列表                            │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  Step 4: 生成华鼎模板                                           │
│  输入：订单数据 + 门店信息 + SKU匹配结果                        │
│  输出：31字段Excel模板                                         │
└─────────────────────────────────────────────────────────────────┘
```

## 五、字段映射规则

### 5.1 门店匹配

| 步骤 | 说明 |
|------|------|
| 1 | 从订单中提取门店名称 |
| 2 | 清理门店名称（去除"河北-"等前缀） |
| 3 | 查询store_list表，匹配门店 |
| 4 | 获取门店编号、仓库、收货人、联系电话、收货地址 |

**失败处理**：
- 使用订单中的门店信息
- 仓库编码使用默认值（5）

### 5.2 SKU匹配

| 步骤 | 说明 |
|------|------|
| 1 | 从订单中提取商品名称 |
| 2 | 查询shipper_sku_mapping表 |
| 3 | 获取系统SKU编号 |
| 4 | 判断单位类型 |

**单位类型判断**：
```python
# SKU配置中有多个单位时，按ratio判断：
# ratio最大的 → 大单位
# ratio中等的 → 中单位
# ratio最小的 → 小单位
```

**失败处理**：
- SKU编号留空
- 备注不填充商品名称（留空）

### 5.3 备注字段来源

⚠️ **重要**：
- 备注字段**必须**从客户订单的"备注"字段提取
- **不是**从商品规格字段填充

## 六、配置要求

### 6.1 必填配置

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **货主ID (shipper_id)** | 标识客户的唯一ID | `HZ2023061500002` |
| **数据库连接** | PostgreSQL连接信息 | host: localhost, port: 5432, database: ai_cs_support |

### 6.2 可选配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **仓库编码映射** | 廊坊仓→5, 天津仓→8 | 从数据库warehouse_code_mapping表加载 |
| **默认仓库编码** | 5 | 门店匹配失败时使用 |
| **输出目录** | `./output` | 生成的模板文件存放目录 |

### 6.3 数据依赖

| 数据 | 说明 |
|------|------|
| **store_list** | 门店列表表，包含 store_code, store_name, warehouse, address, contact_person, phone |
| **shipper_sku_mapping** | SKU映射表，包含 customer_sku_name, system_sku_code, unit_conversion_rule |
| **warehouse_code_mapping** | 仓库编码映射表 |

## 七、错误处理

| 错误情况 | 处理方式 |
|----------|----------|
| 订单文件不存在 | 返回错误：文件不存在 |
| 门店匹配失败 | 使用订单中的门店信息，仓库编码默认5 |
| SKU匹配失败 | SKU编号为空，备注为空 |
| 备注字段为空 | 留空（不是用商品名称填充） |
| 仓库未配置 | 报错退出，提示可用仓库列表 |
