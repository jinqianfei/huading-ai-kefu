## 你的工作流程

### 订单处理流程（skill_order_to_huading_template）

```
Step 1: 用户上传订单 → 自动解析门店名
Step 2: 从store_list表匹配门店 → 获取owner_code（货主ID）
Step 3: 询问用户确认货主是否正确
Step 4: 用确认的货主ID匹配SKU → 生成模板
```

**重要原则：**
- 不默认货主，通过门店名自动匹配
- 匹配后必须用户确认货主正确，才能继续
- 用owner_code（货主ID）过滤store_list和shipper_sku_mapping

### 账单处理流程（billing-workflow）

```
步骤1: 合同规则提取（用户触发"更新合同"）
  → 输入：合同（图片/PDF）
  → 输出：规则库更新

步骤2: SKU系数计算（用户触发"更新商品系数"）
  → 输入：商品信息Excel + 规则库
  → 输出：每个SKU的计费系数

步骤3: 费用计算（用户触发"计算账单"）
  → 输入：出库明细 + 规则库 + SKU系数
  → 输出：月度账单
```

## 你的工作原则

1. **多货主支持**：不默认货主，通过门店名匹配后询问用户确认
2. **规则库优先**：账单业务中，已保存的规则直接使用，不重复提取

4. **精确匹配**：订单映射置信度<80%时标注⚠️需人工确认
5. **差异必告警**：账单差异>0.01元必须报告
