# Skill: 账单制作 (billing-workflow)

**Skill名称**: billing-workflow  
**版本**: v2.3  
**日期**: 2026-05-13  
**用途**: AI账单制作全流程，支持模块化调用

**设计原则**：
- **规则库生命周期**：用户显式触发更新，否则不更新
- **版本管理**：只保留最新版本
- **计算优先**：直接使用已保存规则，无需每次重新提取

---

## 一、核心流程

### 流程图（支持跳过）

```
┌─────────────────────────────────────────────────────────────────┐
│  步骤0：基础配置收集（必须）                                      │
│  输入：客户名称、合同编号、仓库列表、账期                        │
│  输出：基础配置就绪                                               │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  步骤1：规则检查                                                 │
│  检查：规则库中是否有该客户的规则？                              │
│  判断：                                                          │
│    - 用户说"更新合同" → 执行步骤2合同提取                       │
│    - 用户说"更新商品系数" → 执行步骤3系数计算                   │
│    - 用户说"计算账单" → 跳过步骤2-3，进入步骤4                  │
└─────────────────────────────────────────────────────────────────┘
                            ↓（如需更新）
┌─────────────────────────────────────────────────────────────────┐
│  步骤2：合同规则提取（用户显式触发）                            │
│  输入：合同文件（图片/PDF/纸质）                                 │
│  输出：规则库更新                                                 │
│  触发词："更新合同"、"上传新合同"、"解析合同"                   │
└─────────────────────────────────────────────────────────────────┘
                            ↓（如需更新）
┌─────────────────────────────────────────────────────────────────┐
│  步骤3：SKU计费系数计算（用户显式触发）                          │
│  输入：商品信息Excel + 规则库                                    │
│  输出：SKU系数更新到规则库                                       │
│  触发词："更新商品系数"、"重新计算系数"、"更新SKU"               │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  步骤4：AI费用计算（使用已有规则）                                │
│  输入：出库单明细 + 规则库（已保存规则）                         │
│  输出：AI计算的各项费用                                          │
│  公式：                                                          │
│    标准件数量 = 计划出库数量 / 换算率 × 计费系数               │
│    运输费 = max(标准件数量 × 单价, 最低收费)                      │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  步骤5：逐单比对验证                                              │
│  输入：AI计算结果 vs 手动计算Excel账单                           │
│  输出：差异报告                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 用户意图路由

| 用户说 | 执行步骤 | 说明 |
|--------|----------|------|
| "更新合同"、"解析合同" | 步骤2 | 提取合同规则，更新规则库 |
| "更新商品系数"、"重新计算系数" | 步骤3 | 重新计算SKU系数，更新规则库 |
| "制作账单"、"计算账单" | 步骤4-5 | **直接使用已有规则**计算 |
| "全面更新" | 步骤2→3→4→5 | 完整流程 |

## 二、规则库结构

### 存储位置

```
billing-rules/
├── CUST_XXX_contract.json      # 合同规则
├── CUST_XXX_coefficients.json  # SKU计费系数
└── CUST_XXX_latest.json       # 最新版本汇总（含更新时间戳）
```

### 规则库内容

**合同规则 (contract.json)**
```json
{
  "customer_id": "CUST_XXX",
  "customer_name": "客户名称",
  "contract_no": "合同编号",
  "updated_at": "2026-05-12T16:36:00+08:00",
  "updated_by": "客服",
  "rules": {
    "storage": {"常温": 1.0, "冷冻": 3.2},
    "transport": {...},
    "standard_unit": {...}
  }
}
```

**SKU系数 (coefficients.json)**
```json
{
  "customer_id": "CUST_XXX",
  "updated_at": "2026-05-12T16:36:00+08:00",
  "products": {
    "SKU001": {"name": "商品", "coef": 1.0},
    "SKU002": {"name": "商品", "coef": 1.2}
  }
}
```

## 三、各步骤详细说明

### 步骤1：规则检查

| 检查项 | 逻辑 |
|--------|------|
| 是否有合同规则 | 有 → 可跳过步骤2 |
| 是否有SKU系数 | 有 → 可跳过步骤3 |
| 是否需要更新 | 用户说"更新X" → 执行对应步骤 |

### 步骤2：合同规则提取

| 项目 | 说明 |
|------|------|
| 输入 | 合同文件（图片/PDF） |
| 处理 | 视觉LLM解析合同图片，提取计费规则（**v2.4 新增**） |
| 输出 | 更新到 `*_contract.json` |
| 触发确认 | 提取后需客服确认才生效 |

### 步骤3：SKU系数计算

| 项目 | 说明 |
|------|------|
| 输入 | 商品信息Excel（含体积、重量） |
| 处理 | 按合同公式计算每个SKU的计费系数 |
| 输出 | 更新到 `*_coefficients.json` |
| 固定SKU | 合同约定的固定系数优先 |

### 步骤4：AI费用计算

| 项目 | 说明 |
|------|------|
| 输入 | 出库单明细 + 规则库 |
| 处理 | 读取已保存规则 → 计算费用 |
| 输出 | AI计算的账单费用 |
| 公式 | `标准件数量 = 计划出库数量 / 换算率 × 计费系数` |

### 步骤5：逐单比对验证

| 项目 | 说明 |
|------|------|
| 输入 | AI计算结果 vs 客户提供的账单Excel |
| 处理 | 逐单比对差异 |
| 输出 | 差异报告 |

## 四、关键公式

### 标准件数量
```
标准件数量 = 计划出库数量 / 换算率 × 计费系数
```

### 运输费
```
运输费 = max(标准件数量 × 单价, 最低收费)
```

### 计费系数（超规格规则从合同提取）
```
固定SKU：合同约定系数 = 1.0
非固定SKU：
  tier3：超过 tier2 上限 → max(重量/基准重量, 体积/基准体积)
  tier2：合同约定阈值 → 合同约定系数（默认2.0）
  tier1：合同约定阈值 → 合同约定系数（默认1.2）
  标准件：在基准阈值内 → 1.0

注意：tier1/tier2/tier3 的阈值和系数由合同约定，AI 从合同中提取
```
